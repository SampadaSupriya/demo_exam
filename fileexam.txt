Q) Secure VPC Architecture with Bastion Host
Create a VPC named Lab_VPC_<PRN> with CIDR block 10.0.0.0/16. Create one
Public Subnet (10.0.1.0/24) and one Private Subnet (10.0.2.0/24). Attach an
Internet Gateway and configure route tables such that only the Public Subnet
has internet access. Launch one EC2 instance in each subnet. Configure
Security Groups so that the Public EC2 can be accessed from your local
machine through SSH, while the Private EC2 can only be accessed from the
Public EC2. Create a file containing your PRN number on the Public EC2,
transfer it securely to the Private EC2, and verify the contents. Demonstrate
successful SSH connectivity from Local Machine → Public EC2 → Private EC2.
Q) Secure VPC Architecture with Bastion Host
Step 1: Create a Custom VPC
A VPC (Virtual Private Cloud) provides an isolated network environment in AWS. We
create a custom VPC to control networking, routing, and security.
UI Steps
1. Login to AWS Console.
2. Search VPC.
3. Click Your VPCs.
4. Click Create VPC.
5. Select VPC only.
Configuration
Field Value
Name Tag Lab_VPC_<prn>
IPv4 CIDR 10.0.0.0/16
IPv6 CIDR No IPv6
Tenancy Default
6. Click Create VPC.
Explanation of Attributes
●
●
10.0.0.0/16 → Provides 65,536 IP addresses.
Default Tenancy → Uses shared AWS hardware.
Step 2: Create Public Subnet
The Public Subnet will host the Bastion Host (Public EC2) which is accessible from
the internet.
UI Steps
1. Open Subnets.
2. Click Create Subnet.
Configuration
Field Value
VPC Lab_VPC_<prn>
Name Public_Subnet
AZ ap-south-1a
CIDR 10.0.1.0/24
3. Click Create Subnet.
Explanation of Attributes
●
●
10.0.1.0/24 → Provides 256 IP addresses.
Public subnet will receive internet access through IGW.
Step 3: Create Private Subnet
The Private Subnet hosts internal resources which should not be directly accessible
from the internet.
UI Steps
1. Click Create Subnet.
Configuration
Field Value

Name Private_Subnet
AZ ap-south-1b
CIDR 10.0.2.0/24
2. Click Create Subnet.
Explanation of Attributes
●
●
10.0.2.0/24 → Private network segment.
No direct internet access.
Step 4: Create Internet Gateway
Internet Gateway allows internet communication for resources inside the Public

Field Value

5. Click Attach to VPC.

●
IGW enables internet routing.

Field Value

VPC Lab_VPC_
3. Create Route Table.
Add Route

2. Click Route Tables from the left menu.
3. Select Public_RT.
4. Click the Routes tab.
5. Click Edit Routes.
6. Click Add Route.
Fill the details:
Field Value
Destination 0.0.0.0/0
Target Internet Gateway
7. Click the Target dropdown.

Explanation of Attributes
●
●
0.0.0.0/0 → Represents all IPv4 addresses (entire Internet).
Internet Gateway (IGW) → Allows traffic to enter and leave the VPC through

Associate Public Route Table with Public Subnet
What are we doing and why?
A route table becomes effective only after it is associated with a subnet. Associating
Public_RT with Public_Subnet makes that subnet internet accessible.
UI Steps
1. Select Public_RT.
2. Click the Subnet Associations tab.
3. Click Edit Subnet Associations.
4. Select:Public_Subnet (10.0.1.0/24)
5. Click Save Associations.
Verification
After saving, you should see:
Route Table: Public_RT
Associated Subnets:
✓ Public_Subnet
