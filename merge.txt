Q5) Git Branching and Merge Conflict Resolution
Create a Git repository named Lab_Q5 and create three branches named main,
dev1, and dev2. Create a text file named Project_.txt containing 10 lines of content
and commit it to the main branch. Modify the same line(s) of the file differently in
both dev1 and dev2 branches and commit the changes in each branch. Attempt to
merge dev2 into dev1, resulting in a merge conflict. Resolve the conflict manually,
verify the file contents, and complete the merge. Next, make additional modifications
to the same line(s) in both main and dev1 branches and commit the changes.
Attempt to merge dev1 into main, creating another merge conflict. Resolve the
conflict manually, complete the merge, and verify that the final file contains the
intended content from all branches. Finally, display the branch history and merge
history to confirm successful conflict resolution and integration of all changes.
—
